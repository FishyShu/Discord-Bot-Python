Variables & Random Responses

Variables — use these placeholders in your response text or embed fields:
{user} — mentions the user
{username} — the user's display name
{server} — server name
{channel} — channel name
{membercount} — total server members
{args} — everything typed after the command

Random responses — separate multiple responses with --- on its own line. The bot will pick one at random each time.
Example:
Hello {user}!
---
Hey there, {username}! Welcome to {server}.

Trigger types
contains_link — fires when a message contains a URL
contains_file — fires when a message has an attachment
contains_emoji — fires when a message has custom or unicode emoji
contains_role_mention — fires when a message mentions a role
Use Additional Filters to combine conditions (e.g. auto_reply with has_link filter).

Regex patterns — enable "Use Regex" for powerful matching:
\b(hi|hello|hey)\b — matches whole words "hi", "hello", or "hey"
(?i)badword — case-insensitive match (already default)
^\!?\w+ — matches start of message
https?://discord\.gg/\S+ — matches Discord invite links

Multi-trigger — add one pattern per line in "Trigger Patterns". Any match fires the rule.
Priority — higher numbers match first when multiple rules overlap. Default is 0.
